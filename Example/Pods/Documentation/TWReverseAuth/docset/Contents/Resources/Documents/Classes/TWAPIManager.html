+ (void)performReverseAuthForAccount:(ACAccount *)account withHandler:(ReverseAuthResponseHandler)handler

Discussion

Obtains the access token and secret for |account| using either TWRequest or SLRequest.

There are two steps required for Reverse Auth:

The first sends a signed request that you must sign to Twitter to obtain an Authorization: header. You sign the request with your own OAuth keys, which have been granted the Reverse Auth privilege.

The second step uses TWRequest or SLRequest to sign and send the response to step 1 back to Twitter. The response to this request, if everything worked, will include an user’s access token and secret which can then be used in conjunction with your consumer key and secret to make authenticated calls to Twitter.
